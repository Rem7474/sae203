Exercice 5 : gestion dynamique des menus version 2  dans une BDD (FACULTATIF)

Pour faciliter l'administration des menus, notamment par des personnes non habilitées à accéder aux fichiers présents sur le serveur, il est décidé de placer les menus dans une base de données. Cela facilitera le développement d'une interface web d'administration pour la gestion des menus (ajout  suppression  modification).

Dans cet exercice, nous allons placer les menus dans une base de données. Pour des raisons de clarté et de séparation, nous utiliserons une base différente de celle de l'application. Il faudra donc gérer une connexion de base de données supplémentaire. Le connecteur sera nommé $conBDDM.

Créer une base de données votreloginMenus.

A l'aide de la requête ci-dessous, ajouter la table CHAP (IdChap, :texteC, ordreC, urlC)

 

CREATE TABLE CHAP (

idChap INTEGER UNIQUE NOT NULL,

texteC TEXT NOT NULL,            /* dénomination du chapitre */

ordreC INTEGER NOT NULL UNIQUE,  /* ordre d'affichage  correspondra au n° chapitre_en_cours */

urlC TEXT ,                      /* lien url */

groupeC INTEGER,                 /* utilise par la suite */

 

PRIMARY KEY (idChap)

);

 

Le numéro d'ordre des chapitres sera utilisé pour être comparé à la variable '$chapitre_en_cours'.

Insérer quelques chapitres directement dans la base.

Créer le fichier paramConBDDM.php qui contiendra les paramètres de connexion à la base de données des menus.

Et créer/compléter le fichier connexion.php en définissant les fonctions connexionBDDM(), relatif à la nouvelle base.

Les fonctions d'accès à cette base seront placées dans le fichier fonctionBDDM.php.

Analyser le code de Partie2BDD.php qui sera utilisé à la place de Partie2Fichier.php (ne pas renommer les fichiers, mais modifier les 'include').

Ajouter le fichier fonctionBDDM.php.

5.1: menu des chapitres : nom du chapitre en cours surligné

Le style css associé au chapitre en cours est déjà prévu dans le fichier css. Il est nommé : S_Chap. Les autres chapitres n'auront pas de style particulier à préciser Complétez le code du fichier Partie2BDD.php pour qu'il permette l'affichage du menu des chapitres en les surlignant.

 

5.2: menu des chapitres : affichage conditionnel

On souhaite pouvoir afficher ou non un menu suivant l'appartenance à un groupe. Par exemple, lorsqu'un utilisateur appartient à un groupe, certains menus s'affichent et d'autres non.

On considérera qu'un menu ne peut être affiché que pour un groupe. Autrement dit, il correspond à un seul groupe.

Pour simplifier le développement, on forcera l'appartenance à un groupe en utilisant une variable php $groupe_en_cours dans la / chaque page. Cette variable contiendra l'identifiant du groupe en cours.

L'affichage du menu ne se fera que lorsque l'identifiant du groupe en cours est le même que celui qui est présent dans la base de données (champ 'groupec' dans table CHAP).

Faites les modifications nécessaires et tester le résultat.

Vérifier le fonctionnement en ajoutant préalablement des chapitres valides pour un même groupe.

 

5.3: menu des sections : gestion des sections dans une base de données

Tout est dans le titre !

Il vous faudra ajouter la table SECTIONS. La requête est donnée ci-dessous :

CREATE TABLE SECTIONS (

    idsection INTEGER NOT NULL UNIQUE,

    textes TEXT NOT NULL ,

    urls TEXT ,

    refchap INTEGER NOT NULL REFERENCES CHAP ON UPDATE CASCADE ON DELETE RESTRICT,

    ordres INTEGER NOT NULL ,

    PRIMARY KEY (idsection)

);

 

Insérer quelques sections directement dans la base. Le numéro d'ordre des sections sera utiliser pour être comparé à la variable '$section_en_cours'.

Vérifier le fonctionnement.

(Remarque : avec cette solution, rien ne garantit qu'une section d'un même chapitre aient le même ordre).
